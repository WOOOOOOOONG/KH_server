UI(User Interface)란 ?
사람과 사물 또는 시스템, 특히 기계, 컴퓨터 프로그램 등 사이에서 의사소통을 할 수 있도록 일시적 또는 영구적인 접근을 목적으로 만들어진 물리적이고 가상적인 매개체

입력 : 사용자가 시스템을 조작할 수 있게 한다.
출력 : 시스템이 사용자가 이용한 것에 대한 결과를 표시한다.

사용자 인터페이스를 판단하는 기준 : 사용성
- 좋은 사용자 인터페이스는 사용자가 필요한 요소를 쉽게 찾고 사용하며 그 요소로부터 명확하게 의도한 결과를 쉽게 얻어낼 수 있어야 한다.

(참고)UX(User Experience)란 ? 
사용자가 어떤 시스템, 제품, 서비스를 직,간접적으로 이용하면서 느끼고 생각하게 되는 총체적 경험을 말한다.
단순히 기능이나 절차상의 만족 뿐만 아니라 전반적인 지각 가능한 모든 면에서 사용자가 참여, 사용, 관찰하고 상호 교감을 통해서 알 수 있는 가치있는 경험이다.

1. 테스트 기법
- 휴리스틱 평가(Heuristic Evaluation)
사용성에 대한 문제를 찾아내기 위한 사용성 공학 방법으로 전문가에 의해 이론과 경험을 근거로 하여 일련의 규칙들을 만들어 놓고 평가 대상이 규칙을 얼마나 잘 지키고 있는가를 확인하는 기법. (= 전문가가 세워 놓은 규칙에 따르고 있는지)

2. 테스트 계획서
- 테스트의 목표를 달성하기 위해 필요한 활동 내역을 정의한 계획서로, 계획서 항목에는 테스트 계획 ID, 서문, 테스트 항목, 테스트 대상 기능, 테스트 대상 외의 기능, 테스트 방법, 합/불합격 기준, 테스트 시작 조건과 중단 기준, 재개 요건, 성과물, 테스트 작업, 환경, 책임자, 인원 및 교육, 스케쥴, 리스크, 승인 등이 있다.

3. 사용성 테스트 계획서 작성 시 주의할 점
사용성 테스트의 목표, 도구, 방법, 얻고자 하는 결과와 진행절차에 대해서 테스트 계획서에 구체적으로 명시해야 한다.

4. 수동 테스트와 자동 테스트 방식
- UI 자동화 테스트 환경을 구축하는 이유?
UI 자동화 테스트 환경 구축 시 테스트 수행, 평가 및 프로세스 등에서 많은 시간과 비용, 리소스를 감소시킬 수 있고, 동시성, 성능 테스트 등 수동으로는 진행하기 힘든 테스트의 진행이 가능하다.
- Selenium, Naver Guitar 라는 자동화 테스트 툴들이 있음.
